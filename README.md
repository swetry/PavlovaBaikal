# Безопасный SQL Runner
## Отборочное задание для ИТ-школы КОД БАЙКАЛА 2026.

**Описание** <br>
Простой Python-скрипт для безопасного выполнения SQL-запросов к PostgreSQL. Позволяет выполнять только SELECT-запросы с автоматической защитой от SQL-инъекций и превышения лимита выводимых данных.

**Стек технологий**<br>
Python 3.10+<br>
psycopg2-binary — драйвер для PostgreSQL<br>
python-dotenv — работа с переменными окружения<br>

**Требования**<br>
Установленный Python версии 3.10 или выше<br>
Доступ к базе данных PostgreSQL<br>

**Установка**<br>
Клонируйте репозиторий:<br>
git clone <repo-url><br>
cd safe-sql-runner<br>
Создайте виртуальное окружение:<br>
python -m venv venv<br>
source venv/bin/activate  # Linux/macOS<br>
venv\Scripts\activate     # Windows<br>
Установите зависимости:<br>
pip install -r requirements.txt<br>
Настройте переменные окружения:<br>
cp .env.example .env<br>

**Отредактируйте файл .env под свои данные** <br>
Запуск<br>
python main.py<br>
Программа установит соединение с базой данных и перейдет в режим ожидания ввода запросов.<br>

**Примеры использования**<br>
Простой запрос:<br>
sql<br>
SELECT * FROM users;<br>
Скрипт автоматически добавит LIMIT 5.<br>
Запрос с LIMIT:<br>
sql<br>
SELECT name FROM products LIMIT 10;<br>
LIMIT будет использован как указан пользователем.<br>
Запрещенный запрос:<br>
sql<br>
DELETE FROM users;<br>
Скрипт заблокирует выполнение с сообщением об ошибке.<br>

**Выход из программы**<br>
Для завершения работы введите:<br>
exit<br>
quit<br>
q<br>

**Тестирование**<br>
Перед отправкой убедитесь, что:<br>
Скрипт корректно обрабатывает SELECT-запросы<br>
Блокирует опасные команды<br>
Правильно добавляет LIMIT<br>
Корректно работает с интерактивным вводом пароля<br>

**Примечания по безопасности**<br>
Не храните пароль в .env файле<br>
Используйте интерактивный ввод пароля при запуске<br>
Все запросы проверяются на безопасность<br>
